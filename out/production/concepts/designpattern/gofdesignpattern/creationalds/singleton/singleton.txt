1] Database Connections: In applications where creating and managing database connections is an expensive operation,
a Singleton can be used to maintain a single database connection throughout the application.

2] Configuration Management: When you have global configuration settings that need to be accessed
by various components of the application, a Singleton configuration manager can provide a single point of access to these settings.

3] Printing Service: In systems that involve printing documents or reports, a Singleton printing service can coordinate
and manage print jobs, ensuring efficient use of printing resources.
